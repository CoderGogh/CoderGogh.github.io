# 데드락 (DeadLock, 교착 상태)

## 📌 개념

> 두 개 이상의 프로세스나 스레드가 서로 자원을 얻지 못해 다음 처리를 하지 못하고, 무한히 대기하게 되는 상태

마치 **외나무 다리 양 끝에서 서로 비켜주길 기다리는 것**과 같다.

시스템 내 한정된 자원을 여러 프로세스가 동시에 사용하려 할 때 발생한다.

---

## 🔁 데드락 발생 예시

| 시점 | 프로세스 1 | 프로세스 2 |
|------|-----------|-----------|
| t1   | 자원 1 점유 | 자원 2 점유 |
| t2   | 자원 2 대기 | 자원 1 대기 |

→ 서로가 원하는 자원을 상대방이 점유하고 있어 **무한 대기 상태(DeadLock)** 발생

**주로 발생하는 상황:**
- 멀티 프로그래밍 환경에서 한정된 자원을 얻기 위해 프로세스들이 경쟁
- 프로세스가 자원을 요청했을 때 해당 자원을 즉시 사용할 수 없는 경우 → 대기 상태로 진입
- 대기 상태의 프로세스들이 실행 상태로 변경될 수 없을 때 교착 상태 발생

---

## ⚠️ 데드락 발생 조건

> 아래 **4가지 조건이 모두 성립**할 때 데드락이 발생한다.  
> 하나라도 성립하지 않으면 데드락을 예방할 수 있다.

### 1. 상호 배제 (Mutual Exclusion)
자원은 한 번에 **하나의 프로세스만** 사용할 수 있다.

### 2. 점유 대기 (Hold and Wait)
최소 하나의 자원을 점유한 채로, **다른 프로세스가 사용 중인 자원을 추가로 점유하기 위해 대기**하는 프로세스가 존재한다.

### 3. 비선점 (No Preemption)
다른 프로세스에 할당된 자원은 **사용이 끝날 때까지 강제로 빼앗을 수 없다.**

### 4. 순환 대기 (Circular Wait)
프로세스 집합이 **순환 형태로 서로의 자원을 대기**하고 있어야 한다.

---

## 🛠️ 데드락 처리 방법

### 1️⃣ 예방 (Prevention)

발생 조건 4가지 중 **하나를 제거**하여 데드락을 원천 차단한다.  
단, **자원 낭비가 심하다**는 단점이 있다.

| 부정 조건 | 방법 |
|-----------|------|
| 상호 배제 부정 | 여러 프로세스가 공유 자원을 동시에 사용 |
| 점유 대기 부정 | 프로세스 실행 전 필요한 모든 자원을 미리 할당 |
| 비선점 부정 | 자원 점유 중 다른 자원 요구 시 가진 자원 반납 |
| 순환 대기 부정 | 자원에 고유 번호를 할당하고 순서대로만 요구 |

---

### 2️⃣ 회피 (Avoidance)

데드락이 발생할 것 같으면 **미리 피해나가는** 방법

#### 🏦 은행원 알고리즘 (Banker's Algorithm)

- 은행에서 모든 고객의 요구를 충족하도록 현금을 할당하는 것에서 유래
- 프로세스가 자원을 요구할 때, 자원 할당 후에도 **안정 상태(Safe State)** 가 유지되는지 사전 검사
- 안정 상태이면 자원 할당 → 불안정 상태이면 다른 프로세스가 자원 해제할 때까지 대기

---

### 3️⃣ 탐지 (Detection)

데드락이 발생하도록 허용한 후, **자원 할당 그래프**를 통해 탐지한다.  
자원 요청 시마다 탐지 알고리즘을 실행하므로 **오버헤드**가 발생할 수 있다.

---

### 4️⃣ 회복 (Recovery)

탐지 후 교착 상태를 해제하는 방법

**프로세스 종료 방법:**
- 교착 상태의 프로세스를 **모두 중지**
- 교착 상태가 제거될 때까지 **하나씩 프로세스 중지**

**자원 선점 방법:**
- 교착 상태 프로세스가 점유 중인 자원을 **강제로 선점**하여 다른 프로세스에 할당 (해당 프로세스는 일시 정지)
- 우선순위가 낮거나 수행 횟수가 적은 프로세스 위주로 선점

---

## 🍽️ 식사하는 철학자 문제 (Dining Philosophers Problem)

**기아 상태(Starvation)** 를 설명하는 대표적인 문제

- N명의 철학자가 원형 테이블에 앉아 있고, 각자 양쪽 젓가락을 모두 들어야 식사 가능
- 모두 동시에 왼쪽 젓가락을 들면 아무도 오른쪽을 들지 못해 **데드락 발생**

**해결책:**
1. N명이 앉을 수 있는 테이블에 **N-1명만 앉힌다**
2. 젓가락 두 개를 **모두 집을 수 있는 상황에서만** 집도록 허용
3. 일부 철학자는 **오른쪽 젓가락을 먼저** 집도록 허용 (순환 방지)

## ✅ 요약 정리

| 구분 | 설명 |
|------|------|
| 개념 | 프로세스들이 서로 자원을 기다리며 무한 대기 |
| 발생 조건 | 상호 배제, 점유 대기, 비선점, 순환 대기 (4가지 모두 충족 시 발생) |
| 예방 | 발생 조건 중 하나를 제거 (자원 낭비 큼) |
| 회피 | 은행원 알고리즘으로 안정 상태 유지 |
| 탐지 | 자원 할당 그래프로 데드락 탐지 |
| 회복 | 프로세스 종료 또는 자원 선점 |
