## 들어가며

반도체 Fab은 사람이 거의 들어가지 않는 무인 자동화 공장이다. 웨이퍼가 담긴 캐리어(FOUP)가 공정 설비 사이를 끊임없이 이동해야 하는데, 이 전체 물류 시스템을 **AMHS(Automated Material Handling System)**라고 부른다. 이번 글에서는 AMHS의 전체 구조를 먼저 잡고, 그 안에서 실제로 FOUP을 나르는 하드웨어인 **OHT**와 이를 지휘하는 소프트웨어인 **MCS**를 정리한다.

## 1. Fab과 AMHS 기본 개념

### 1.1 왜 자동반송이 필요한가

반도체 공정은 한 장의 웨이퍼가 수백~수천 개의 공정 단계(노광, 식각, 증착, 세정 등)를 거치며 짧게는 수 주, 길게는 수 개월에 걸쳐 완성된다. 이 과정에서 웨이퍼는 외부 오염(파티클)에 극도로 취약하기 때문에, 사람이 직접 들고 옮기는 방식은 사용할 수 없다.

그래서 웨이퍼는 **FOUP(Front Opening Unified Pod)**이라는 밀폐 용기에 담겨 이동하며, 이 FOUP을 설비와 설비 사이로 옮기는 역할을 AMHS가 전담한다.

### 1.2 Fab의 공간 구조

Fab 내부는 일반적으로 아래와 같은 계층으로 나뉜다.

```
┌─────────────────────────────────────────┐
│              Sub-Fab (지하층)             │
│   - 배관, 전력, 가스, 화학약품 공급 설비    │
└─────────────────────────────────────────┘
┌─────────────────────────────────────────┐
│            Fab 층 (Clean Room)            │
│                                           │
│   [Bay A]──[Bay B]──[Bay C]──[Bay D]      │
│      │ Inter-Bay 레일로 연결               │
│      └── 각 Bay 내부는 Intra-Bay 레일      │
│                                           │
│   천장: OHT 레일망 (전체를 그물처럼 연결)   │
└─────────────────────────────────────────┘
```

- **Bay**: 동일하거나 유사한 공정 설비들을 모아둔 구역 단위
- **Intra-Bay 반송**: 같은 Bay 안에서의 짧은 거리 이동
- **Inter-Bay 반송**: 서로 다른 Bay 사이의 장거리 이동
- **Stocker**: FOUP을 임시 보관하는 자동 창고. Bay 입구나 레일 교차 지점에 배치되어 버퍼 역할을 한다

### 1.3 AMHS 구성 요소

AMHS는 하나의 장비가 아니라 여러 하위 시스템의 집합이다.

| 구성 요소 | 역할 |
|---|---|
| OHT (Overhead Hoist Transport) | 천장 레일을 달리며 FOUP을 직접 운반하는 차량 |
| Stocker | FOUP 임시 보관용 자동 창고 |
| MCS (Material Control System) | OHT 차량들에게 작업을 할당하고 경로를 지휘하는 소프트웨어 |
| MES (Manufacturing Execution System) | 전체 생산 공정을 관리하며 MCS에 반송 명령을 내리는 상위 시스템 |
| Conveyor / RGV 등 | OHT를 보조하는 지상/저층 반송 수단 (Fab에 따라 선택적으로 사용) |

명령이 내려오는 흐름은 다음과 같다.

```
MES (공정 스케줄 관리)
   │  "FOUP #1234를 Etch 설비 → CVD 설비로 이동시켜라"
   ▼
MCS (차량 할당 + 경로 계산 + 트래픽 제어)
   │  "OHT #57번, 이 경로로 가서 픽업/이송하라"
   ▼
OHT (실제 주행, 호이스트 동작 수행)
```

## 2. OHT 하드웨어 구동 원리

OHT는 천장에 설치된 고정 레일을 따라 주행하며 FOUP을 정확하게 집어올리고 내려놓는 자율주행 로봇이다.

```
[ 천장 고정 레일 (HID 전력 케이블 매립) ]
              ││
       ┌──────┴┴──────┐
       │   OHT 대차    │ ───▶ 주행/조향 모터, 센서, 무선 통신
       └──────┬┬──────┘
              ││ (벨트/로프)
       ┌──────┴┴──────┐
       │   디스탠서   │ ───▶ 승강 및 좌우 슬라이딩 구동
       └──────┬┬──────┘
            [FOUP]
```

### 2.1 무접촉 전원 공급 (HID, Hot In Dynamic)

OHT는 배터리를 충전하기 위해 멈추지 않는다. 천장 레일에 매립된 HID 케이블에 고주파 전류를 흘려보내 발생하는 전자기 유도 현상을 이용해, 이동 중에도 무선으로 전력을 실시간 공급받는다.

### 2.2 주행 및 분기(조향) 메커니즘

대차 내부의 서보 모터가 구동 바퀴를 굴려 레일 위를 시속 15~20km 속도로 고속 주행한다. 레일이 갈라지는 분기점에 도달하면 가이드 롤러나 전용 조향 링크가 물리적으로 전환되어 선로를 바꾼다.

### 2.3 호이스트(Hoist) 및 슬라이딩 승강

목적지 장비 상단에 멈추면 내부에 감겨 있던 고강도 벨트(또는 로프)가 풀리면서 파지 장치(Gripper)가 아래로 내려간다. 장비 입구가 레일 정중앙에 위치하지 않을 때를 대비해, 승강부가 측면으로 이동하는 좌우 슬라이딩(Side-Shift) 기능을 갖춰 정밀하게 위치를 보정한다.

### 2.4 안전 및 정밀 위치 제어 센서

- 레이저 센서와 바코드/RFID 리더를 통해 선행 OHT 차량과의 충돌을 방지한다.
- 설비 포트 위에 정확히 FOUP을 정렬한다 (±1mm 이내).
- 고속 가감속 시 웨이퍼에 충격이 가해지지 않도록 능동형 진동 제어(Anti-sway) 알고리즘이 모터 출력에 실시간 반영된다.

## 3. MCS 소프트웨어의 핵심 제어 알고리즘

MCS는 상위 생산관리시스템(MES)으로부터 "A 설비에서 B 설비로 FOUP을 이동하라"는 명령을 전달받은 뒤, 수백 대의 OHT를 일사불란하게 지휘하는 소프트웨어다.

### 3.1 차량 할당 알고리즘 (Vehicle Assignment)

- **가까운 차량 우선 (Shortest Distance)**: 명령이 발생한 설비 위치와 가장 가까운 곳에 있는 비어 있는(Empty) OHT를 호출한다.
- **목적지 기반 매칭 (Look-Ahead)**: 특정 장비로 이동 중인 OHT가 하차 직후 근처에서 발생하는 다음 작업을 바로 수행할 수 있도록 연쇄 매칭하여 무부하(빈 차 상태) 주행 거리를 최소화한다.

### 3.2 동적 경로 탐색 알고리즘 (Dynamic Routing)

기본적으로 네트워크의 최단 경로를 찾는 다익스트라(Dijkstra) 알고리즘이나 A* 알고리즘을 활용한다. 공장 내 상황은 실시간으로 변하므로 레일 구간별 주행 비용(Cost)을 실시간 업데이트한다. 특정 구간에 OHT 밀집도가 높아지면 해당 레일의 가중치(Cost)를 높여 후속 OHT들이 정체 구간을 알아서 우회하도록 실시간 가이드한다.

### 3.3 데드락 방지 알고리즘 (Deadlock Prevention)

단방향 혹은 루프 구조의 레일 망에서 OHT 차량들이 서로 꼬리를 물고 맞물려 움직이지 못하는 교착 상태(Deadlock)를 예방한다. 레일 교차로나 분기점을 통과할 때, 특정 구역(윈도우/블록)에 동시 진입 가능한 차량 수를 엄격히 제한(예: 은행원 알고리즘 기반 변형 제어)하여 병목과 충돌을 원천 차단한다.

> 이 부분은 이전에 정리했던 OS의 데드락 4가지 조건 및 은행원 알고리즘과 본질적으로 같은 문제다. 다만 프로세스/자원이 아니라 OHT 차량/레일 구간 단위로 적용된다는 점이 다르다.

### 3.4 저장 공간 연계 스케줄링 (Stocker & Buffer Coordination)

목적지 생산 설비가 이미 가동 중이어서 FOUP을 바로 받을 수 없는 상태(Port Full)가 감지되면, MCS는 경로를 변경한다. 주변의 천장 버퍼(Side Track Buffer)나 스토커(자동창고)에 임시 임치하도록 가이드하여 레일 위 대기 차량으로 인한 교통 정체를 막는다.

## 4. 네 가지 알고리즘이 함께 동작하는 흐름

```
MES 명령 수신
   │
   ▼
[차량 할당] 가장 가까운/효율적인 OHT 선정
   │
   ▼
[경로 탐색] Dijkstra/A* + 실시간 Cost로 최적 경로 계산
   │
   ▼
[데드락 방지] 교차로/분기점 진입 허용량 체크
   │ (제한 초과 시 대기 또는 우회)
   ▼
[버퍼 연계] 목적지 Port Full 여부 확인
   │ (Full이면 Stocker/Buffer로 우선 이동)
   ▼
OHT 실제 주행 및 FOUP 이송 수행
```

## 5. AMHS 관련 핵심 지표 (KPI)

| 지표 | 의미 |
|---|---|
| Delivery Time | 명령 발생부터 FOUP이 목적지에 도착하기까지 걸리는 시간 |
| Vehicle Utilization | 전체 OHT 중 실제로 FOUP을 싣고 운반 중인 차량의 비율 |
| Empty Travel Ratio | 빈 차 상태로 이동한 거리의 비율 (낮을수록 효율적) |
| Throughput | 단위 시간당 처리 가능한 FOUP 반송 건수 |
| Port Blocking Time | 목적지 Port가 막혀 OHT가 대기한 시간 |

## 정리

- **AMHS**는 OHT, Stocker, MCS, MES가 유기적으로 연결된 Fab 전체 물류 시스템이다.
- **OHT**는 HID 무접촉 급전, 고속 주행/분기, 호이스트·슬라이딩 승강, Anti-sway 제어로 FOUP을 안전하게 운반하는 하드웨어다.
- **MCS**는 차량 할당, 동적 경로 탐색, 데드락 방지, 버퍼 연계 스케줄링이라는 네 가지 알고리즘을 조합해 수백 대의 OHT를 실시간으로 지휘한다.
- 특히 데드락 방지 알고리즘은 OS 교착 상태 이론(은행원 알고리즘 등)이 산업 자동화 시스템에 그대로 응용되는 좋은 사례다.
